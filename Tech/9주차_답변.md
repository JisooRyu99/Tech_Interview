## #1
### Ridge와 Lasso에 대해 설명해주세요.
Ridge는 선형 회귀 모델에서 L2 규제를 주는 것으로 회귀계수가 상대적으로 큰 변수의 민감도를 줄임으로써 해당 변수의 값이 많이 달라지는 new sample들의 추정치에 대한 overfit 위험을 줄이는 방법입니다. Lasso는 선형 회귀 모델에서 L1 규제를 주는 것으로 불필요한 변수의 회귀계수를 0으로 만들어 해당 변수를 제외시키는 결과를 얻게 됩니다.
</br>

## #2
### Feature selection 방법을 설명해주세요.
output을 예측하는데에 상관이 없는 변수들이 존재한다면 overfitting이 발생할 수 있습니다. Feature Selection을 하면 학습 시간을 줄일 수 있고, 모델의 분산을 줄임으로써 보다 robust하게 학습되며 모델이 간소화가 되면서 결과 해석이 더 쉬워지는 장점이 있습니다.
Feature Selection의 주된 목적은 독립 변수중에서 중복되거나 종속변수 y와 관련없은 변수들을 제거하여 y를 가장 잘 예측하는 변수들의 존합을 찾아내는 것입니다. Feature Selection에는 크게 3가지 방법이 있습니다. Feature의 조합을 바꿔가며 기계학습을 돌리고 성능이 가장 좋은 조합을 찾아내는 Wrapper method, 전처리 과정에서 미리 Feature selection을 통계적 방법으로 실행하고 모델을 적합하는 방법인 Filter Method, Lasso Regression, Ridge Regression, Decision Tree등의 알고리즘을 통해 feature Selection을 수행하는 Embedded method입니다.
3가지 방법중에서 가장 많이 사용되는 방법은 Filter method입니다.

## #3
### 히스토그램과 막대그래프의 차이점을 설명해주세요
막대그래프는 범주로 구분되는 데이터를 표현하는데 사용하므로 막대로 표현하려는 범주의 순서는 의도에 따라 바뀔 수 있으며 막대 간에는 일정한 간격을 유지합니다. 반면에 히스토그램은 측정된 연속적인 값으로 표시되는 데이터를 표현하는데 사용하므로 막대그래프와 같이 막대의 순서를 임의적으로 바꿀 수 없으며, 막대 간의 간격 없이 표현합니다.

</br>

## #4
### "상관관계는 인과관계를 의미하지 않는다"라는 말이 있습니다. 설명해주실 수 있나요?
하나의 요인으로 인해 다른 요인의 수치가 변하는 형태처럼, 원인과 결과 관계가 명확한 것을 인과관계라고 합니다. 
반면에 상관관계는 두 변수가 얼마나 상호 의존적인지를 의미합니다. 인과관계는 상관관계에서 원인과 결과의 관계까지 명확히 밝히지만, 상관관계만으로는 인과관계를 정확히 밝히지는 못합니다.
> 예시
> - 키와 몸무게는 일정한 상관관계가 있지만 키가 크다고 반드시 몸무게가 많이 나간다고 할 수 없기 때문에 상관관계가 있다고 인과관계가 있다고 할 수 없다.
>

## #5
### Context Switching이 무엇인지 설명해주세요.
Context Switching이란 여러 프로세스를 처리해야 하는 상황에서 현재 진행중인 Task(프로세스 혹은 쓰레드)의 상태를 PCB에 저장하고, 다음에 진행할 Task의 상태 값을 읽어 레지스터에 적재하는 과정을 말합니다.
CPU는 하나의 프로세스만 처리할 수 있습니다. 컴퓨터를 사용할 때 게임을 하면서 음악을 듣고 인터넷 검색을 하는 것은 마치 여러개를 동시에 수행하고 있는 것 같지만 CPU에서 인지하지 못할 정도로 빠르게 프로세스 사이를 왔다갔다 처리를 하면서 하니씩 처리합니다. 이 떄 일어나는 것이 Context Switching입니다.

</br>

## #6
### Gil의 한계점을 보완할 수 있는 방법을 설명해주세요.(비동기 방식에 대해 더 자세하게, 비동기와 멀티 프로세싱이 어떻게 다른 것인지)
GIL을 획득한 스레드만이 인터프리터에서 실행될 수 있기 때문에 멀티스레딩 작업임에도 불구하고 작업 중인 특정 스레드 때문에 다른 모든 스레드들이 멈춰야 하는 특성은 싱글 스레드 작업방식과 차이가 없기 때문에 멀티프로그래밍 특유의 효율성이 사라져 버리게 됩니다. 이를 해결하기 위해 Thread 대신 multi-processing을 사용하는 방법이 있습니다. 각각의 프로세스는 자신만의 메모리가 존재하기 때문에 GIL이 문제가 되지 않습니다.

## #8
### 멤버함수, 정적함수에 대해 설명해주세요.
클래스 내에서 정의된 변수를 **멤버변수**라고 하며 self. 와 함께 사용할 수 있습니다. 멤버 변수는 같은 클래스의 인스턴스들과 같을 공유하기 때문에 초기화되지 않습니다. 클래스 내부에서 사용하기 위해서는 클래스 이름.변수명으로 사용할 수 있습니다.
**정적 함수**는 클래스 내에서 @staticmethod 데코레이터를 사용하여 정의됩니다. 정적 함수내에서는 인스턴스 메서드와 달리 첫번째 매개 변수가 할당되지 않습니다. 따라서 정적 메서드 내에서는 인스턴스/클래스 속성에 접근하거나, 인스턴스/클래스 메서드를 호출하는 것이 불가능합니다.

## #9
### Generator란 무엇인가요?
제네레이터 함수는 iterator(반복)와 같은 루프의 작용을 컨트롤하기 위해 쓰여지는 특별한 함수입니다. 제너레이터는 모든 값을 포함한 배열이나 리스트를 return하는 함수와 비슷하지만 return 구문 대신 yield를 사용하여 한 번 호출될 때마다 하나의 값만을 리턴합니다. 

> 제너레이터는 일반 iterator에 비해 아주 작은 메모리를 필요로합니다. 그래서 실행 시간 보다 메모리 소비를 줄여야 하는 경우라면 제너레이터를 사용해야하고, 리소스 보다 실행 시간을 줄여야하는 경우라면 리스트를 사용해야합니다.
