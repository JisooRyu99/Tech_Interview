## #3
### backpropagation시 더하기, 곱하기, 활성화함수 등의 각 노드에서 어떻게 연산되는지?
덧셈일 경우 $z = x+y$의 미준은 $x$에 대해 미분해도 1, $y$에 대해 미분해도 1이기 떄문에 덧셈 노드의 역전파는 입력 값을 그대로 플려보냅니다.
곱셈일 경우 $z = xy$의 미분은 $x$에 대해 미분하면 $y$, $y$에 대해 미분하면 $x$가 됩니다. 
활성화 함수의 역전파는 하류?애서 들어온 값에 미분값을 고한 값을 상류?로 보냅니다. 활성화 함수에는 ReLu, Sigmoid와 같이 여러 종류가 있습니다.
- ReLu : 0 if x > 0 else 1 (순전파 때의 입력인 x가 0보다 크면 값을 보내고, x가 0이하면 역전파 때는 값을 보내지 않음)
- Sigmoid : y(1-y) (순전파의 값 y만을 이용해, 순전파 값에 y(1-y)를 곱해서 하류로 신호를 보냄)

## #4
### zero-centered가 아닐 경우 무슨 문제가 발생하나요?
input의 값이 항상 양수 혹은 음수라면 gradient W는 항상 같은 방향으로 움직이게 됩니다. 그렇게 되면 직선(대각선)으로 해를 탐색하는 것이 아닌 **zig zag**로 해룰 탐색하게 되는데,
이는 여러 번 탐색을 해야 하기 때문에 비효율적인 weight update 방법입니다. 이러한 이유로 일반적으로 zero-mean data를 원하고 input x가 양수/음수를 모두 가지고 있으면
gradient w가 전부 Positive/Negative로 움직이는 것을 막을 수 있습니다.

## #5
### 인덱스는 크게 Hash 인덱스와 B-Tree 인덱스가 있습니다. 이것은 무엇일까요?
**B-Tree 인덱스**
- equal(=)뿐만 아니라 >, >=, <, <= 또는 between 연산자 사용 가능
- B-Tree는 항상 정렬된 상태를 유지하기 때문에 조회 속도는 빠르지만 삽입과 수정, 제거 작업은 느리다.
> - B-Tree는 정렬된 트리이므로 탐색이 $O(logN)$의 시간 복잡도를 갖는다.
  
**Hash 인덱스**
- 동등 비교 검색에는 최적화돼 있지만 범위를 검색하거나 정렬된 결과를 가져오지는 못한다.
- DBMS에서 해시 인덱스는 메모리 기반의 테이블에 주로 구현되어 있으며 디스크 기반의 데용량 테이블로용으로는 거의 사용되지 않는다.
- 정렬할 필요가 없으니 삽입/삭제가 빠를 수 있다.

## #7
### 힙(heap)이 동적할당을 구현하는데 사용되는 이유가 무엇인가요?
python은 다른 언어와 다르게 동적 할당의 기능이 없어 사용자가 직접 메모리 할당 범위를 조정하지 않습니다. (파이썬이 자동으로 메모리를 관리해주는 언어이기 때문)
Heap 메모리 영역은 사용자의 동적 할당으로 생성되는 공간이며, 사용자가 공간의 크기를 직접 관리할 수 있습니다. 

> python memory manager는 힙 메모리에 있는 객체를 참조하는 형태로 동적할당을 알아서 해준다. interpreter가 포인터를 사용해서 힙 메모리 영역의 범위를 조정하고,
자동으로 OS로 할당하지 않고 저장하고 있다가, 메모리가 필요할 때마다 할당한다. 이렇게 OS랑 소통하면서 결과값이 리턴되거나 변수와 함수의 사용이 멈췄을 때 메모리가 소멸된다.

</br>

## #8
### Garbage Collector(GC)가 무엇인가요?
OS는 프로그램을 프로세스로 실행하게 되고, 해당 프로세스는 메모리에 code, heap, stack 영역을 할당받게 됩니다. 이 때, heap, stack에 할당된 메모리들을 해제하는 역할을 Garbage Collector가 수행하게 됩니다.
Python에서의 Garbage Collecting은 Reference counting, Generational garbage collection가 있습니다.
</br>

Reference counting
- python 객체의 reference count는 객체가 참조될 때마다 증가하고 객체의 참조가 해제될 대 감소한다. 객체의 Reference count가 0이 되면 객체의 메모리 할당이 해제된다.

Generational garbage collection
- 어떤 동적 객체가 서로를 참조(순환참조)하고 있다면 Reference counting하는 방식에서는 개게들을 메모리 해제될 수 없다. 이러한 경우를 방지하기 위해 Generational garbage collection이 순환 참조를 탐지하고 메모리에서 해제해준다.

</br>
